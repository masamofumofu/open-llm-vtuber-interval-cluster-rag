# 階層的区間クラスタリングRAGの数学的定式化

## 要旨

本稿では、小規模なローカル言語モデルおよび運用知識ベースを対象として、階層的区間クラスタリングRAG（Hierarchical Interval Clustering RAG; HIC-RAG）を数学的に定式化する。中心的な考えは、検索結果を単なる点スコアとして扱うのではなく、意味表現、語彙信号、出典範囲、信頼度を含む区間値の根拠対象として表し、それらを凝集型階層クラスタリングによって組織化することである。各クラスタは区間包絡とクラスタ半径を持ち、デンドログラム上の粗密探索、信頼度ゲート、根拠付き生成を支える。本稿では、区間距離、枝刈りのための許容下界、検索スコア・クラスタ凝集度・出典被覆度を組み合わせた信頼度関数を定義する。主結果として、階層クラスタがコーパスの入れ子被覆をなすこと、許容下界による枝刈りが上位候補を保存すること、根拠制約付き生成が形式的な幻覚リスクを非増加に抑えることを示す。最後に、Open-LLM-VTuber、Ollama、TinySwallowを用いたローカルLLM構成における試作結果との対応を述べる。

## 1. はじめに

検索拡張生成（Retrieval-Augmented Generation; RAG）は、言語モデルのパラメータ内部に保存された知識だけに依存せず、外部文書を検索し、その根拠に基づいて回答を生成する枠組みである。特にローカルLLMや小規模モデルでは、モデル単体の事実記憶やプロジェクト固有知識の保持能力が限られるため、RAGは実用上の中核機構となる。

標準的なRAGでは、文書チャンクを埋め込み空間上の点として表し、クエリに近い上位チャンクを取得し、それらを生成器へ渡すことが多い。この構成は有効である一方で、検索スコアの不確実性、検索結果間の階層的関係、根拠集合としてのまとまりを明示的には表現しない。すなわち、個々のチャンクが高スコアであっても、それらが同一の説明単位を形成しているか、出典範囲が十分に連続しているか、根拠として信頼できるかは別問題である。

本稿では、これまで検討してきた階層的区間クラスタリングRAGを数学的に定式化する。基本方針は三つである。第一に、チャンクを点ではなく区間値対象として扱う。第二に、それらの区間値対象に対して凝集型階層クラスタリングを行い、根拠領域のデンドログラムを構成する。第三に、検索時にはデンドログラムを粗いクラスタから細かいチャンクへ探索し、検索スコア、クラスタ半径、出典被覆度、候補間の曖昧性を組み合わせて回答可否を判定する。

本定式化は、Open-LLM-VTuber、Ollama、TinySwallow、および設定ファイル・ログ・実装ファイルを対象とするローカル知識ベースの試作に基づく。これまでの試作では、設定、ログ、エラー、ファイルに関する質問をRAGへ振り分け、挨拶や雑談を通常LLM応答へ回すRAGゲートを実装した。本稿は、その工学的成果を数学的な枠組みとして整理するものである。

## 2. 準備

### 2.1 コーパス、チャンク、検索

文書コーパスを

```text
D = {d_1, ..., d_N}
```

とする。各文書はチャンクへ分割され、チャンク集合を

```text
C = {c_1, ..., c_n}
```

と書く。各チャンク `c` は、本文、文書ID、ファイル名、行範囲またはページ範囲、更新時刻などのメタデータを持つ。クエリ `q` に対して、RAG検索器は回答根拠となる候補集合 `E_k(q)` を返す。

通常の密ベクトル検索では、クエリとチャンクのスコアに基づき、

```text
R_k(q) = arg top-k_{c in C} score(q,c)
```

が計算される。本稿では、この点スコア表現を区間値根拠対象へ拡張する。

### 2.2 区間値根拠対象

チャンク `c` の埋め込みを `phi(c) in R^m` とする。通常は `phi(c)` を一点として扱うが、本稿ではチャンクを次の区間値対象として表す。

```text
X(c) = (I_1(c), ..., I_m(c), L(c), S(c), R(c)).
```

ここで、`I_j(c) = [a_j(c), b_j(c)]` は第 `j` 次元の意味表現の不確実性区間、`L(c)` はBM25等に由来する語彙スコア区間、`S(c)` は出典範囲、`R(c)` は検索信頼度区間である。なお、本稿の基本距離では `R(c)` を直接の距離成分には含めず、後述する信頼度関数における補助情報として用いる。信頼度を距離そのものへ組み込む拡張では、`eta delta_R(X,Y)^2` のような項を追加できる。

最も単純には、

```text
I_j(c) = [phi_j(c) - eps_j(c), phi_j(c) + eps_j(c)]
```

と定める。半径 `eps_j(c)` は、チャンク長、近傍密度、埋め込みモデル間の分散、検索器間の不一致、検索スコアの揺らぎなどから推定できる。短く曖昧なクエリでは区間半径を大きくし、設定キー、関数名、エラー文、ファイルパスなどを含む具体的なクエリでは区間半径を小さくする。

### 2.3 区間距離

区間 `A = [a,b]` に対し、中心と半径を

```text
mid(A) = (a+b)/2,
rad(A) = (b-a)/2
```

と定義する。二つの区間 `A, B` の距離を

```text
delta_I(A,B)^2 = (mid(A)-mid(B))^2
               + lambda (rad(A)-rad(B))^2,  lambda >= 0
```

とする。二つの区間値根拠対象 `X, Y` の距離は、

```text
d(X,Y)^2 =
  sum_{j=1}^m w_j delta_I(I_j(X), I_j(Y))^2
  + alpha delta_L(X,Y)^2
  + beta  delta_S(X,Y)^2
```

で定義する。`delta_L` は語彙スコア区間の距離、`delta_S` は出典範囲の距離である。同一ファイル内の近接範囲では `delta_S` を小さくし、別ファイルまたは異なる文脈に属する範囲では大きくする。

`R(c)` を距離から分離する理由は、意味的近さと検索結果としての採用可否を区別するためである。すなわち、二つのチャンクが意味的に近くても、根拠として採用できるかどうかは、検索器の一致、出典の連続性、候補間の曖昧性などを合わせて判断する。

## 3. 階層的区間クラスタリング

### 3.1 区間包絡とクラスタ半径

クラスタ `A subset C` に対し、その区間包絡 `H(A)` を各次元で次のように定義する。

```text
H_j(A) = [ min_{c in A} a_j(c), max_{c in A} b_j(c) ].
```

語彙スコア、信頼度区間、出典範囲についても同様に包絡または集約区間を定義する。ここで `H(A)` は単なる点代表ではなく、クラスタ `A` を代表する区間値対象として扱う。クラスタ半径を

```text
rho(A) = max_{c in A} d(X(c), H(A))
```

とする。`rho(A)` はクラスタ内部の意味的ばらつきを表す。RAGにおいては、クラスタ半径が小さいほど、根拠チャンク群が一貫した説明単位を形成しているとみなせる。

### 3.2 凝集型階層化

初期状態では、各チャンクを一つのクラスタとする。

```text
P_0 = {{c_1}, ..., {c_n}}.
```

各ステップで二つのクラスタ `A, B` を選び、`A union B` に併合する。併合基準には平均連結、完全連結、Ward型基準などを用いることができる。本稿では、根拠集合の一貫性を重視するため、次のWard型増分を基本形とする。ただし、出典距離や語彙距離を含む一般の `d` は厳密なユークリッド距離とは限らない。そのため、以下の基準は、`d` がユークリッド表現に埋め込める場合には通常のWard基準として解釈でき、そうでない場合にはWard型ヒューリスティックとして用いる。

```text
Delta(A,B) = SSE(A union B) - SSE(A) - SSE(B),
SSE(A) = sum_{c in A} d(X(c), mu_A)^2.
```

ここで `mu_A` はクラスタ `A` の代表区間中心である。`Delta(A,B)` が小さい組を順に併合することで、根拠クラスタのデンドログラム `T` が得られる。

## 4. HIC-RAG検索

### 4.1 粗密探索

クエリ `q` も区間値対象 `X(q)` として表す。クエリの意味区間 `I_j(q)` と語彙区間 `L(q)` は、クエリ表現およびクエリ変種から構成する。一方、通常の自然言語クエリは出典範囲を持たないため、`S(q)` は明示的に指定されたファイル名、設定キー、関数名、ログ範囲などがある場合にのみ定義する。出典指定がない場合は、`beta=0` として出典距離項を無効化するか、全出典を含む広い区間として `S(q)` を置く。検索はデンドログラム `T` の上位クラスタから開始し、クラスタ `A` とクエリ `q` の下界距離を計算する。

```text
lb(q,A)^2 =
  sum_j w_j dist(I_j(q), H_j(A))^2
  + alpha dist(L(q), L(A))^2
  + beta  dist(S(q), S(A))^2.
```

ここで `dist` は二つの区間が交差する場合は0、交差しない場合は区間間の最短距離である。下界が大きいクラスタは探索対象から除外し、下界が小さいクラスタのみを展開する。最終的にチャンク単位の候補を得て、通常の再ランキングを行い、根拠集合 `E_k(q)` を得る。

### 4.2 信頼度関数

検索結果の信頼度を、検索スコアだけでなくクラスタのまとまりと出典被覆度から定義する。

```text
Rel(q,E) =
  sigma(gamma_0 + gamma_1 s(q,E)
        - gamma_2 rho(E)
        + gamma_3 cov(E)
        - gamma_4 amb(E)).
```

ここで、`s(q,E)` は検索スコア、`rho(E)` は根拠集合を含む最小クラスタの半径、`cov(E)` は必要範囲の連続的な出典被覆度、`amb(E)` は上位候補と次点候補の差が小さい場合に大きくなる曖昧性ペナルティである。`sigma` はロジスティック関数である。例えば、上位候補と次点候補の生の差を `raw_gap(E)` とすると、`amb(E)=1/(raw_gap(E)+epsilon)` のように定められる。したがって、候補間の差が小さいほど `amb(E)` は大きくなり、信頼度は下がる。

`Rel(q,E) < tau` の場合、システムは回答を控える、追加確認を求める、または「根拠が不足しています」と返すべきである。

## 5. 主結果

### 定理1（入れ子被覆）

凝集型階層クラスタリングにより得られる任意時点のクラスタ族 `P_t` は `C` の分割である。また、時点が進むにつれて分割列は入れ子構造をなす。すなわち、任意の `A in P_t` と後続時点 `s > t` に対して、ある `B in P_s` が存在し、`A subset B` を満たす。

**証明.** 初期時点では単集合族 `{{c_1}, ..., {c_n}}` が `C` の分割である。各ステップでは互いに素な二つのクラスタ `A, B` を `A union B` に置き換え、他のクラスタはそのまま保存する。したがって分割性は帰納的に保たれる。また、各クラスタは保存されるか、より大きなクラスタに吸収されるだけである。よって任意の前時点クラスタは後続時点のあるクラスタに含まれる。

### 補題2（区間下界の許容性）

クラスタ `A` と任意の `c in A` に対して、下界距離 `lb(q,A)` は

```text
lb(q,A) <= d(X(q), X(c))
```

を満たす。

**証明.** 任意の次元において、`X(c)` の区間はクラスタ包絡 `H(A)` に含まれる。したがって、クエリ区間から包絡区間への最短距離は、クエリ区間から個別チャンク区間への距離以下である。語彙区間および出典範囲についても同様である。非負重み付き二乗和を取り、平方根を適用しても不等式は保存される。

### 定理3（上位候補保存枝刈り）

探索中の暫定第 `k` 位距離を `r_k` とする。あるクラスタ `A` が

```text
lb(q,A) > r_k
```

を満たすなら、`A` に含まれるどのチャンクも現在の上位 `k` 件を改善しない。したがって、`A` を枝刈りしても最終的な上位 `k` 候補は変わらない。

**証明.** 補題2より、任意の `c in A` について `d(X(q),X(c)) >= lb(q,A) > r_k` が成り立つ。よって `A` 内の全チャンクは暫定第 `k` 位より遠く、上位 `k` 件に入らない。

### 命題4（根拠制約付き生成による幻覚リスクの非増加）

生成器 `G` が、根拠集合 `E` に支持されない主張を出力しないという制約を満たすと仮定する。クエリ `q` に答えるために必要な事実集合が `E` に含まれる事象を `A_q` とする。幻覚リスク `H` を「根拠に支持されない主張を出力する確率」として測るなら、同一の基礎生成器に根拠制約を付加した系 `G` と、その制約を持たない系 `G_0` の間で、

```text
H(G | A_q) <= H(G_0 | A_q)
```

が成り立つ。ここでの主張は、制約が理想的に満たされる場合の形式的な非増加性であり、実際のLLMが常にこの条件を完全に満たすことを意味しない。

**証明.** `A_q` のもとでは、回答に必要な事実は `E` に含まれる。`G` は `E` に支持されない主張を出力しないため、非根拠主張の発生確率は0で上界される。一方、非制約生成器 `G_0` は `E` 外の主張を生成しうる。したがって条件付き幻覚リスクは増加しない。実運用では、この理想的制約を、引用照合、文単位の支持判定、回答拒否閾値によって近似する。

## 6. 試作実装との対応

上記の数学的対象は、これまでのローカルLLM試作と次のように対応する。ただし、現行実装は本稿の定式化をすべて完全に実装したものではなく、TF-IDF検索、クエリ変種、簡略化した区間距離、階層クラスタリング、RAGゲート、短文TTS化を組み合わせた試作である。本稿の定式化は、今後のembeddingモデル化、信頼度推定、出典被覆度評価を含む拡張形を与える。

| 段階 | 入力 | 出力 | 目的 |
|---|---|---|---|
| 区間化 | チャンク、埋め込み、メタデータ | 根拠対象 `X(c)` | 検索不確実性を保持する |
| 階層化 | 区間値対象集合 | デンドログラム `T` | 根拠領域を組織化する |
| 検索 | クエリ `X(q)` | 候補集合 `E_k(q)` | 粗密探索を行う |
| 信頼度判定 | スコア、半径、被覆度 | `Rel(q,E)` | 回答可否を判定する |
| 生成 | クエリと根拠 | 根拠付き回答 | 非根拠主張を抑制する |

現在のOpen-LLM-VTuber構成では、RAGゲートがクエリの種別を判定する。設定、ログ、エラー、ファイル関連の質問はRAGへ振り分けられ、挨拶や雑談は通常LLM応答へ回される。これは、クエリ `q` をRAG検索空間に写すべきかどうかを判定する前処理である。

また、OllamaとTinySwallowを用いた回答生成テストでは、上位クラスタへ文脈を絞り、強化されたクエリをLLMへ渡すことで、設定項目に関する質問に対して信頼度の高い根拠チャンクを利用した日本語回答が得られた。これは、信頼度関数において `s(q,E)` を高め、`rho(E)` を小さくする実装上の対応例である。

## 7. 考察

階層的区間クラスタリングRAGの意義は、単に検索を高速化することではない。より重要なのは、RAGにおける「根拠らしさ」を数学的対象として扱える点である。通常の上位 `k` チャンク検索では、個々のチャンクが高スコアでも、それらが相互に一貫した根拠集合を形成しているとは限らない。これに対して、階層クラスタの半径、包絡、出典被覆度を用いれば、根拠集合が一つの説明単位としてまとまっているかを評価できる。

一方で課題も残る。第一に、区間半径 `eps_j(c)` の推定方法は経験的に調整する必要がある。第二に、Ward型基準は大規模コーパスでは計算量が大きくなるため、近似最近傍探索や定期的なオフライン再構築との統合が必要である。また、一般の距離を用いる場合はWard型基準をヒューリスティックとして扱う必要がある。第三に、命題4は理想化された生成制約を仮定している。実際には、生成文ごとの根拠照合、引用単位の検証、回答拒否基準を組み合わせる必要がある。

## 8. 結論

本稿では、階層的区間クラスタリングRAGを、区間値データ解析、凝集型階層クラスタリング、検索拡張生成を結合する枠組みとして定式化した。チャンクを区間値根拠対象として扱い、それらをデンドログラム上に組織化することで、候補検索、根拠集合化、信頼度判定、根拠付き生成を一貫した数式で記述できる。

今後の課題は、実際の検索ログから区間半径を推定し、クラスタ半径と回答品質の関係を測定し、信頼度関数が非根拠回答や根拠不足回答をどの程度予測できるかを検証することである。

## 付録A. 擬似コード

```text
Algorithm HIC-RAG(q, T, k, tau)
1. if Gate(q) = false:
       return NormalLLM(q)
2. X(q) <- IntervalizeQuery(q)
3. traverse dendrogram T using lb(q,A)
4. rerank candidate chunks and obtain E_k(q)
5. compute Rel(q,E_k)
6. if Rel(q,E_k) < tau:
       return evidence-insufficient response
7. return GenerateWithEvidence(q,E_k)
```
